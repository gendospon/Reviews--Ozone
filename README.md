# Claude-Ozon — Автоматизация ответов на отзывы Ozon

CLI-утилита для работы с отзывами маркетплейса Ozon через Seller API. Позволяет получать необработанные отзывы, составлять ответы по правилам тональности компании и отправлять их на Ozon — всё через командную строку с поддержкой dry-run режима.

## Что умеет

- **Получать необработанные отзывы** — `fetch` вытягивает из Ozon все отзывы со статусом `UNPROCESSED` с пагинацией
- **Отвечать на отзывы** — `post` отправляет ответ на конкретный отзыв (по умолчанию в dry-run, без реальной отправки)
- **Помечать отзывы обработанными** — `mark-processed` переводит отзыв в статус `PROCESSED`
- **Автоматические ретраи** — при 429 (rate limit) и 500-х ошибках скрипт повторяет запрос
- **Защита от случайной отправки** — флаг `--live` требуется явно, без него ответ только выводится на экран

## Установка и настройка

### 1. Клонирование и зависимости

```bash
git clone https://github.com/gendospon/Reviews--Ozone.git
cd claude
pip install -r requirements.txt
```

Требуется Python 3.8+.

### 2. API-ключи Ozon

Создайте файл `.env` в корне проекта:

```
OZON_CLIENT_ID=<ваш Client-ID>
OZON_API_KEY=<ваш Api-Key>
```

Ключи берутся в кабинете продавца Ozon: https://seller.ozon.ru → Настройки → API-ключи.

**Важно:** файл `.env` добавлен в `.gitignore` и никогда не попадает в Git. Не коммитьте ключи и не делитесь ими.

### 3. Проверка подключения

```bash
python ozon_reviews.py fetch
```

Если ключи верные — скрипт выведет список необработанных отзывов. При HTTP 401/403 — проверьте ключи в `.env`. При 429 — превысили лимит запросов, подождите.

## Команды

images/start.jpg

```bash
# Получить необработанные отзывы (до 50 по умолчанию)
python ozon_reviews.py fetch
python ozon_reviews.py fetch --limit 10


# Ответить на отзыв (dry-run — ответ не отправляется)
python ozon_reviews.py post <review_id> "Текст ответа"


# Ответить на отзыв (реальная отправка на Ozon)
python ozon_reviews.py post --live <review_id> "Текст ответа"


# Пометить отзыв как обработанный
python ozon_reviews.py mark-processed <review_id>
```

## Установка на другой компьютер через Claude Code

Чтобы любой сотрудник мог работать с проектом, достаточно Claude Code и Git.

### Предварительные требования

- [Python 3.8+](https://www.python.org/downloads/)
- [Git](https://git-scm.com/downloads)
- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) — CLI-ассистент от Anthropic

### Шаги установки

1. **Клонируйте репозиторий:**

   ```bash
   git clone https://github.com/gendospon/Reviews--Ozone.git
   cd claude
   ```

2. **Установите зависимости:**

   ```bash
   pip install -r requirements.txt
   ```

3. **Создайте файл `.env`** с вашими ключами Ozon (см. раздел «API-ключи Ozon» выше).

4. **Откройте проект в Claude Code:**

   ```bash
   claude
   ```

   Claude Code автоматически обнаружит файл `CLAUDE.md` и скил `/project` — он уже знает структуру проекта, правила тональности и команды CLI.

5. **Используйте скил для работы с отзывами:**

   Введите в Claude Code:

   ```
   /project Обработай отзывы
   ```

   Claude подключится к Ozon API, получит необработанные отзывы и составит ответы по правилам из `brand_voice.md`.

## Правила тональности ответов

Ответы на отзывы формулируются по правилам из `brand_voice.md`. Кратко:

| Рейтинг | Подход |
|---------|--------|
| 4-5 (позитивные) | Поблагодарить от лица коллектива, пригласить зайти ещё |
| 3 (нейтральные) | Поблагодарить, выразить сожаление, признать проблему, пообещать исправить |
| 1-2 (негативные) | Глубоко посочувствовать, поблагодарить за критику, пообещать учесть |
| Без текста | Ответить по рейтингу + вежливо попросить поделиться мнением |

Общий тон: дружелюбный, с лёгким юмором, от лица команды. Никакой агрессии, оправданий или канцелярита.

## Безопасность

- `.env` исключён из Git — ключи никогда не попадают в коммиты
- Команда `post` без флага `--live` работает в dry-run режиме — ответ не отправляется на Ozon
- `post --live` требует явного указания — Claude всегда предупреждает перед реальной отправкой
- Между запросами к API соблюдается задержка 1 секунда (`REQUEST_DELAY = 1.0`)
