# Claude-Ozon — Автоматизация ответов на отзывы Ozon

Проект для работы с отзывами маркетплейса Ozon через Seller API.

## Структура проекта

```
ozon_reviews.py   — CLI-утилита: получение отзывов, отправка ответов, смена статуса
brand_voice.md    — Правила тональности ответов (обязательны к соблюдению при составлении ответов)
.env              — API-ключи Ozon (НЕ коммитить, НЕ показывать в логах и ответах)
requirements.txt  — Зависимости Python
```

## Подключение к Ozon Seller API

Ключи хранятся в файле `.env` в корне проекта. Формат:

```
OZON_CLIENT_ID=<твой Client-ID из кабинета Ozon>
OZON_API_KEY=<твой Api-Key из кабинета Ozon>
```

**Как получить ключи:**
1. Войти в личный кабинет продавца Ozon → https://seller.ozon.ru
2. Настройки → API-ключи (или Настройки → Интеграция с API)
3. Создать новый ключ или скопировать существующий
4. Вставить значения в `.env` файл

**Безопасность:**
- Файл `.env` добавлен в `.gitignore` и никогда не попадает в Git
- Никогда не выводи значения `OZON_CLIENT_ID` и `OZON_API_KEY` в логах, ответах или файлах
- Если ключи не заданы — скрипт выведет ошибку и завершится с кодом 1

## Установка и запуск

```bash
pip install -r requirements.txt
```

Python 3.8+.

## Команды CLI

```bash
# Получить необработанные отзывы (по умолчанию до 50)
python ozon_reviews.py fetch
python ozon_reviews.py fetch --limit 10

# Ответить на отзыв (dry-run — безопасный режим по умолчанию)
python ozon_reviews.py post <review_id> "Текст ответа"

# Ответить на отзыв (реальная отправка на Ozon)
python ozon_reviews.py post --live <review_id> "Текст ответа"

# Пометить отзыв как обработанный
python ozon_reviews.py mark-processed <review_id>
```

## Правила работы с отзывами

При составлении ответов на отзывы всегда следуй правилам из `brand_voice.md`. Ключевые принципы:

- **Позитивные (4-5):** поблагодарить от лица коллектива, пригласить зайти ещё раз
- **Нейтральные (3):** поблагодарить, выразить сожаление, признать проблему и пообещать исправить
- **Негативные (1-2):** глубоко посочувствовать, поблагодарить за критику, пообещать учесть замечания
- **Без текста:** ответить по рейтингу + вежливо попросить поделиться мнением
- **Тон:** дружелюбный, с лёгким юмором, от лица команды. Никакой агрессии, оправданий или канцелярита

## Безопасность при отправке ответов

- `post` без флага `--live` — это dry-run, ответ не отправляется на Ozon
- `post --live` — реальная отправка. Всегда предупреждай пользователя перед использованием `--live`
- Никогда не отправляй ответы в live-режиме без явного подтверждения пользователя

## Внесение изменений

- Основной код — `ozon_reviews.py`. Это единственный Python-файл проекта
- При добавлении новых API-эндпоинтов Ozon используй функцию `ozon_post()` — она уже обрабатывает ретраи, rate limiting и ошибки авторизации
- `BASE_URL = "https://api-seller.ozon.ru"` — все эндпоинты идут от этого базового URL
- Задержка между запросами: `REQUEST_DELAY = 1.0` секунда (уважай rate limit Ozon)

## Проверка ошибок

```bash
python -c "import ozon_reviews"   # проверить, что модуль импортируется
python ozon_reviews.py fetch       # проверить подключение к API (нужны ключи в .env)
```

Если при `fetch` получаешь HTTP 401/403 — ключи в `.env` неверные или устарели.
Если получаешь 429 — превышен лимит запросов, подожди и повтори.
