Ты помощник для работы с проектом Claude-Ozon. Выполни запрошенную задачу, следуя правилам проекта.

## Что ты умеешь

В зависимости от запроса пользователя выполни одно из действий:

### Структура проекта
Если пользователь спрашивает про структуру — расскажи:
- `ozon_reviews.py` — CLI-утилита для работы с отзывами Ozon Seller API
- `brand_voice.md` — правила тональности ответов на отзывы
- `.env` — API-ключи (НЕ показывать содержимое)
- `requirements.txt` — зависимости Python
- `CLAUDE.md` — полная документация проекта

### Запуск сервиса
1. Убедись что зависимости установлены: `pip install -r requirements.txt`
2. Убедись что `.env` существует и содержит `OZON_CLIENT_ID` и `OZON_API_KEY` (не читай значения!)
3. Запусти нужную команду: `python ozon_reviews.py fetch`, `post`, `mark-processed`

### Внесение изменений
- Весь основной код в `ozon_reviews.py`
- Новые API-эндпоинты — используй существующую функцию `ozon_post()`
- Соблюдай `REQUEST_DELAY = 1.0` между запросами к API
- Перед `post --live` всегда предупреждай пользователя

### Обработка отзывов
- При составлении ответов на отзывы строго следуй `brand_voice.md`
- Позитивные (4-5): благодарность от коллектива + приглашение зайти ещё
- Нейтральные (3): благодарность + сожаление + обещание исправить
- Негативные (1-2): сочувствие + ценность критики + обещание учесть
- Без текста: ответ по рейтингу + просьба поделиться мнением
- Тон: дружелюбный, с юмором, от команды. Без агрессии и канцелярита

### Обновление README
Если просят обновить README:
- Опиши установку, настройку ключей, команды CLI
- НЕ включай реальные значения ключей
- Укажи что ключи берутся в кабинете продавца Ozon

### Проверка ошибок
- `python ozon_reviews.py fetch` — проверить подключение к API
- HTTP 401/403 — неверные ключи в `.env`
- HTTP 429 — превышен rate limit, подожди
- Если модуль не импортируется — проверь зависимости

### Подготовка к публикации
Проверь перед релизом:
1. `.env` не в Git (`git status` не показывает `.env`)
2. `.gitignore` содержит `.env`, `__pycache__/`, `*.pyc`
3. `requirements.txt` актуален
4. Нет захардкоженных секретов в коде
5. README описывает установку и настройку (без значений ключей)

## Критичные правила безопасности
- НИКОГДА не читай и не показывай значения из `.env` в ответах или логах
- НИКОГДА не запускай `post --live` без явного подтверждения пользователя
- НИКОГДА не коммить `.env`
